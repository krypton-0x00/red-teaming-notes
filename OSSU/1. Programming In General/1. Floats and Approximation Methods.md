## **Part 2 â€“ Integers in Binary**

Integers are stored exactly.

**What it does**:

`/ `â†’ true division, result is a float (even if divisible exactly).

```python

# // â†’ floor division, result is the largest integer â‰¤ the exact division result.

```
Example: `num = 6`:

* `6 % 2 = 0`, num â†’ 3
* `3 % 2 = 1`, num â†’ 1
* `1 % 2 = 1`, num â†’ 0
* Binary (reverse order) = `110`.

Integers are **straightforward** â€” no rounding issues.

## **Part 4 â€“ When it fails**

Some fractions (like `1/10`) never terminate in binary:

* `0.1` in binary = `0.000110011001100110011...` (repeats forever)
* We **cut it off** after a fixed number of bits â†’ rounding.

## **Part 10 â€“ Approximation method**

Goal: approximate `sqrt(x)`.

```python
x = 36
epsilon = 0.01
num_guesses = 0
guess = 0.0
increment = 0.0001

while abs(guess**2 - x) >= epsilon:
    guess += increment
    num_guesses += 1

print('num_guesses =', num_guesses)
print(guess, 'is close to square root of', x)
```

Explanation:

* Start at 0.0, increase by small steps.
* Stop when `guess**2` is within `epsilon` of `x`.
* **Trade-off**:

  * Small `increment` â†’ accurate but slow.
  * Large `increment` â†’ fast but less accurate.

## **Part 11 â€“ Overshooting fix**

We can skip over the â€œclose enoughâ€ zone entirely:

```python

    guess += increment
```

Stops if:

* Youâ€™re close enough, OR
* Youâ€™ve gone past the target.
